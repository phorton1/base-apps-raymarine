### Corrections from Patrick during review (full set)

**Sections 2–3:**

1. **Homogeneous restriction is E80-paste side only.** DB copy/cut has no type
   restriction. Any DB selection (including mixed) produces a clipboard and can
   be pasted to DB. The restriction lives at the E80 paste destination only.

2. **Groups are compound objects on E80.** Pasting a group to E80 carries the
   group shell (UUID, name) AND its member waypoints as a compound operation.
   Pre-flight must check both levels: group UUID/name conflict AND member waypoint
   conflicts. Create order: group shell first, then member waypoints inside it.
   Not a monadic operation.

3. **Route points are NOT waypoints.** They are ordered UUID references defining
   a route's sequence. Right-click on a route point in the DB tree shows only:
   Delete (labeled "Delete" in UI; implementation is REMOVE_ROUTEPOINT splice),
   and Paste Before/After (from a compatible waypoints clipboard). No Copy, no Cut,
   no separate clipboard type for route points.

4. **Clipboard is a container, not an enumerable typed vocabulary.** The vocabulary
   table (section 3 of first version) was fundamentally wrong. The clipboard holds:
   source, cut flag, items list, selection-origin (branch?). It has no type label.
   At paste time it answers queries: homogeneous? what type? source? cut?
   `[concept notion]` identifiers (e.g., `[copy waypoints from DB]`) belong in the
   test plan as shorthand for specific test setups — they are NOT clipboard type
   labels in the design doc.

5. **E80 delete has no UI-level blocks.** Unlike DB (which blocks "delete WP in
   route" with an informational message), the E80 panel never blocks. Pre-flight
   detects dependencies (waypoint in a route not being deleted), warns user.
   Execution order is always: DELETE_ROUTE first, then DELETE_GROUP/DELETE_GROUP_WPS,
   then DELETE_WAYPOINT.

6. **My Waypoints is an E80-only reserved pseudo-group name.** No DB group may
   be named "My Waypoints." E80→DB download from My Waypoints creates individual
   ungrouped waypoints in the target DB collection — no group is created.

7. **Paste Before / Paste After / Paste New Before / Paste New After** — four new
   positional paste operations. Available on any member node (not collection root).
   DB side: clipboard type must be compatible with destination node type; waypoints
   clipboard + route point destination inserts into route at that position.
   E80 side: clipboard must be truly homogeneous AND same type as destination node.
