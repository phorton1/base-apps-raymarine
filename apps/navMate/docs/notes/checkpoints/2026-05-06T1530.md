## Fix: editor Save does not re-render visible items on Leaflet map

When saving changes in the winDatabase editor (color, name, lat/lon, etc.),
the Leaflet map was not updated — the stale feature remained.

### Fix (winDatabase.pm, _onSave)

Four lines added before `disconnectDB` and `$this->refresh()`:

```perl
if ($type eq 'object' && $rendered_uuids{$uuid})
{
    _pullFromLeaflet($this, $uuid);
    _pushObjToLeaflet($dbh, $this, { uuid => $uuid, obj_type => $obj_type });
}
```

If the saved item is currently rendered (in `%rendered_uuids`), the stale
feature is removed from Leaflet and a fresh one is pushed from DB. Items
not currently on the map are unaffected.
