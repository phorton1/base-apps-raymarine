For groups and routes, items carry `members => [...]`.

## e_wp_api.pm API (package d_WPMGR, continued — file: NET/e_wp_api.pm)

Get service handles:
```perl
my $wpmgr = $raydp->findImplementedService('WPMGR');
my $track = $raydp->findImplementedService('TRACK');
```
(helpers `_wpmgr()` and `_track()` in nmOps.pm)

WPMGR calls:
```perl
$wpmgr->createWaypoint({name, uuid, lat, lon, sym=>0, ts, comment, depth, progress})
$wpmgr->modifyWaypoint({uuid, name, lat, lon, sym=>0, ts, comment, depth, progress})
$wpmgr->deleteWaypoint($uuid, $progress, $waypoint_only)  # $waypoint_only=1 skips _removeFromGroup
$wpmgr->createGroup({name, uuid, comment=>'', members=>[], progress})
$wpmgr->modifyGroup({uuid, members=>\@uuids, progress})
$wpmgr->deleteGroup($uuid, $progress)
$wpmgr->setWaypointGroup($wp_uuid, $group_uuid, $progress)  # undef/$group_uuid = My Waypoints
$wpmgr->createRoute({name, uuid, comment=>'', color=>$idx, waypoints=>\@uuids, progress})
$wpmgr->modifyRoute({uuid, waypoints=>\@uuids, progress})
$wpmgr->deleteRoute($uuid, $progress)
$wpmgr->routeWaypoint($route_uuid, $wp_uuid, $add, $progress)  # queues GET_ITEM automatically
$wpmgr->findUUIDByName($what_name, $name)  # $what_name: 'waypoint'|'group'|'route'
```

LAT/LON BOUNDARY: createWaypoint/modifyWaypoint accept decimal degrees; they scale to 1e7 internally.
E80 wpmgr memory stores lat/lon as 1e7-scaled ints.
When reading from wpmgr memory for clipboard: divide by $SCALE_LATLON before storing.

TRACK calls:
```perl
$track->queueTRACKCommand($apps::raymarine::NET::d_TRACK::API_GENERAL_CMD, $uuid, 'erase')
```

## Progress dialog pattern

```perl
my $progress = Pub::WX::ProgressDialog::newProgressData($total);
$progress->{active} = 1;
my $dlg = Pub::WX::ProgressDialog->new(getAppFrame(), $title, 1, $progress);
return if !$dlg;
# For operations that trigger GET_ITEM callbacks (routes, groups):
```

Check `$progress->{cancelled}` in loops. All Paste-to-E80 and Delete-from-E80 use this pattern.

## Key breaks from old code to new

1. **No `intent` in clipboard.** Old clipboard had `intent => 'waypoint'|'group'|...`.
   New clipboard is raw `{source, cut_flag, items[]}`. All classification in pre-flight.

2. **Unified COPY/CUT constants.** Old: ~9 per-type COPY + ~9 per-type CUT.
   New: one COPY=10010, one CUT=10110.

3. **`onContextMenuCommand` moves.** Old: lived in nmClipboard.pm.
   New: lives in nmOps.pm. nmClipboard.pm is state + menu generation only.

4. **No `_deconflictE80Name`.** Old code silently renamed on E80 name collision.
   New: hard-abort with message (SS10.2 Steps 6 and 7). User must rename before retrying.

5. **Pre-flight is a first-class function in nmOps.pm** running before ALL operations
   (Delete, Copy, Cut, Paste). Not just E80 writes.

6. **PASTE_BEFORE/AFTER** — entirely new; route-point sequence insertion on both panels.
   E80 only supports before/after at route-point destinations (SS10.9).

7. **Version fields** — `db_version` and `e80_version` maintained on every DB write path.
   See SS12.4 for the full table of rules.

8. **Position FLOAT assignment** — new items must get a position value.
   Pattern: `SELECT COALESCE(MAX(position),0)+1 FROM <table> WHERE <scope>=?`

9. **Route dependency check before E80 route paste** — every member WP UUID must
   already exist in WPMGR. No silent side-effect WP creation (SS10.10).

## Behavioral rules (critical)

- NEVER `die` or `kill` in .pm files — use `error("msg"); return;`
- `warning(level, indent, msg)` — three args; `error(msg)` — one arg
- Edit pre-existing files: detect tabs vs spaces first; use Perl :raw if Edit tool fails
- Never Write tool on pre-existing files (changes line endings)
- `use` order: pure Perl -> Wx/Pub -> NET -> own local modules -> use base
- Always explicit `qw()` imports from Pub::Utils; never full namespace
- Schema guard: confirm before any SQLite schema/structure change
- No polling loops; no `Yield()` in polling
