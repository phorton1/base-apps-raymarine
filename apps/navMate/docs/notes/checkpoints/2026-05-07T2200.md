use Wx qw(:everything);

**Private IDs for conflict dialog:** `$ID_REPLACE=10901`, `$ID_SKIP=10902`,
`$ID_REPLACE_ALL=10903`, `$ID_SKIP_ALL=10904`, `$ID_ABORT=10905`.

**`_refreshDatabaseWithDelete(@uuids)`** — calls `$pane->onObjectsDeleted(@uuids)` then
`$pane->refresh()`. Used after every delete that removes specific objects from the DB.
Plain `_refreshDatabase()` (no UUIDs) is still used for operations where WPs survive
(e.g. DELETE GROUP where WPs move to parent).

**`_resolveConflict($tree, $title, $item_name)`** — Wx::Dialog with 5 buttons:
Replace / Skip / Replace All / Skip All / Abort. Returns action string. Veto on close.
EVT_BUTTON calls EndModal(GetId()).

**`_wpFieldsDiffer($existing, $new_data, $source)`** — compares name/comment/lat/lon always;
also wp_type/color/depth_cm if source eq 'database'. Lat/lon: if abs > 900, divides by 1e7
(handles E80 int encoding). Returns 1 if any field differs.

**`_insertFreshWaypoint($dbh, $coll_uuid, $wp, $source)`** — no UUID arg → fresh UUID.
ts_source set to 'e80' if source eq 'e80', else wp->{ts_source}//'user'.
Handles depth: `$wp->{depth_cm} // $wp->{depth} // 0`.

**`_pasteOneWaypointToDB($dbh, $coll_uuid, $tree, $item, $source, $policy_ref, $title)`**
UUID-preserving. Returns: 'created'/'replaced'/'skipped'/'no_change'/'aborted'.
Policy chain: replace_all→replace, skip_all→skip, else _resolveConflict dialog.
Updates $$policy_ref to sticky values (replace_all/skip_all/abort).

**Cut helpers** (all open own DB connection):
- `_cutDatabaseWaypoint($uuid, $tree)` — checks route ref count first; blocks if > 0
- `_cutDatabaseGroup($uuid, $tree)` — blocks if remaining members > 0
- `_cutDatabaseRoute($uuid, $tree)` — always deletes
- `_cutDatabaseTrack($uuid, $tree)` — always deletes

**`_deleteDB($cmd_id, $right_click_node, $tree, @nodes)`** — dispatch:
- `CTX_CMD_REMOVE_ROUTEPOINT` — confirmDialog, removeRoutePoint, _refreshDatabaseWithDelete(uuid)
- `CTX_CMD_DELETE_WAYPOINT` — check route refs (IMPL ERROR if any), confirmDialog, deleteWaypoint x N, _refreshDatabaseWithDelete
- `CTX_CMD_DELETE_GROUP` — get group info (parent_uuid+WPs), confirmDialog with WP count,
  moveWaypoint each WP to parent, deleteCollection, _refreshDatabase (no UUIDs — WPs survive)
- `CTX_CMD_DELETE_GROUP_WPS` — check route refs per WP (IMPL ERROR if any), confirmDialog,
  deleteWaypoint each + deleteCollection, _refreshDatabaseWithDelete(wp_uuids)
- `CTX_CMD_DELETE_ROUTE` — single: get WP count for msg; multi: generic msg; deleteRoute x N,
  _refreshDatabaseWithDelete
- `CTX_CMD_DELETE_TRACK` — confirmDialog, deleteTrack x N, _refreshDatabaseWithDelete
- `CTX_CMD_DELETE_BRANCH` — confirmDialog, getCollectionWRGTs for UUID list, deleteBranch,
  _refreshDatabaseWithDelete

**`_pasteItemsToCollection($dbh, $items, $target_uuid, $source, $fresh, $cut_flag, $tree, $policy_ref)`**
Recursive helper for all collection paste types. Called by _pasteDB and by itself for
branches/groups.

Per item type:
- `waypoint`: fresh→_insertFreshWaypoint; cut+DB→moveWaypoint; else _pasteOneWaypointToDB
  + optional cut (_cutDatabaseWaypoint or _cutE80Waypoint stub)
- `group`: fresh→insertCollection(GROUP)+recurse members with fresh=1;
  cut+DB→moveCollection; else insertCollectionUUID if not exists, recurse members
  with _pasteOneWaypointToDB, optional _cutDatabaseGroup/_cutE80Group if not aborted+not skipped
- `route`: fresh→insertFreshWaypoint each rp + insertRoute + appendRouteWaypoint;
  cut+DB→moveRoute; else _pasteOneWaypointToDB each rp with own route_policy,
  then insertRouteUUID/updateRoute + clearRouteWaypoints + appendRouteWaypoint per rp,
  optional _cutDatabaseRoute/_cutE80Route
- `track`: cut+DB→moveTrack; DB source + non-fresh → IMPL ERROR (no UUID-preserving DB copy);
  else insertTrack + insertTrackPoints (with depth/temp field name mapping), optional cut
- `branch`: fresh→insertCollection(BRANCH)+recurse members fresh=1;
  cut+DB→moveCollection; else insertCollectionUUID if not exists + recurse members
  (non-fresh, same cut_flag)

**`_pasteDB($cmd_id, $right_click_node, $tree, $items, $cb)`**
- `$fresh` = (cmd_id in PASTE_NEW, PASTE_NEW_BEFORE, PASTE_NEW_AFTER)
- PASTE_BEFORE/AFTER branch: flatten items to waypoints (expand group members);
  get route_uuid+insert_pos from right_click_node; get route->collection_uuid;
  UPDATE route_waypoints SET position+=N WHERE position>=insert_pos; insert/paste WPs;
  appendRouteWaypoint at insert_pos+i; deferred cut after DB work; _refreshDatabase
- Standard paste: get target_uuid from right_click_node->data->uuid;
  connectDB; _pasteItemsToCollection; disconnectDB; _refreshDatabase

## What step 5 (nmOpsE80.pm) needs to implement

### Functions to replace stubs:

**`_pasteE80($cmd_id, $right_click_node, $tree, $items, $cb)`**
The $items list is ALREADY pre-flighted (ancestor-wins, branch-dissolved, homogeneous,
name-collision-checked) by nmOps::_doPaste. $cb has source/cut_flag.
cmd_id is one of: PASTE, PASTE_NEW, PASTE_BEFORE, PASTE_AFTER, PASTE_NEW_BEFORE, PASTE_NEW_AFTER.

PASTE/PASTE_NEW to E80:
- Waypoints: createWaypoint via WPMGR; PASTE_NEW = always create; PASTE = check UUID first
- Groups: createGroup, then createWaypoint for each member
- Routes: createWaypoint for each route_point (if not exists), createRoute, addRouteWaypoint
- Tracks: createTrack + addTrackPoints (via TRACK service not WPMGR)

PASTE_BEFORE/AFTER is route-point specific:
- right_click_node is a route_point (uuid, route_uuid, position in E80 array)
- Insert WPs at that position in the E80 route

Cut handling: if cb->cut_flag and source='database', call _cutDatabaseWaypoint etc.
(already implemented in nmOpsDB.pm). If source='e80', defer to _cutE80* after paste.

**`_deleteE80($cmd_id, $right_click_node, $tree, @nodes)`**
Dispatch:
- CTX_CMD_DELETE_WAYPOINT → deleteWaypoint via WPMGR for each
- CTX_CMD_DELETE_GROUP → deleteGroup (waypoints stay as ungrouped)
- CTX_CMD_DELETE_GROUP_WPS → deleteWaypoint each member, then deleteGroup
- CTX_CMD_DELETE_ROUTE → deleteRoute (waypoints stay)
- CTX_CMD_REMOVE_ROUTEPOINT → removeRoutePoint from E80 route
- CTX_CMD_DELETE_TRACK → deleteTrack via TRACK service

All E80 deletes go through the async command queue and use ProgressDialog.
After E80 delete completes, refresh E80 panel.

**`_newE80Group($right_click_node, $tree)`**
Show a TextEntryDialog, createGroup via WPMGR.

**`_newE80Route($right_click_node, $tree)`**
nmDialogs::showNewRoute, createRoute via WPMGR.

**`_newE80Waypoint($right_click_node, $tree)`**
nmDialogs::showNewWaypoint + parseLatLon, createWaypoint via WPMGR.

**`_cutE80Waypoint/Group/Route/Track`** — delete from E80 after successful paste to DB.
These are the cut-source cleanup after a cut-paste from E80 → DB.
deleteWaypoint/deleteGroup/deleteRoute/deleteTrack via WPMGR/TRACK.
