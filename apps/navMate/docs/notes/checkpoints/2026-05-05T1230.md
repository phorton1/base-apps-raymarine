## Session 2026-05-05T1230 — generic window editors design decisions

Answers to the four open questions from checkpoint 2026-05-05T1215.md.

### Decision 1 — Scope: winDatabase first, winE80 later

winDatabase is the starting point. winE80 gets the same treatment eventually
for symmetry, but is out of scope for this implementation pass.

### Decision 2 — Editable fields

Rule: any non-structural field on a DB record that could reasonably be edited.
Structural = uuid, collection_uuid, parent_uuid, point counts, timestamps,
ts_source, source. These are display-only even in the property grid.

By type:

  Collection:   name, comment
                (node_type is structural — defines the collection kind)

  Waypoint:     name, comment, lat, lon, wp_type, color, sym, depth_cm
                (uuid, collection_uuid, created_ts, ts_source, source → not editable)

  Route:        name, comment, color
                (uuid, collection_uuid → not editable)
                (route waypoint list shown in text panel below, not editable here)

  Route point:  no editor — editing route points not currently available

  Track:        name, color
                (not much else makes sense to edit via this UX)
                (uuid, collection_uuid, ts_start, ts_end, ts_source, point_count → not editable)

### Decision 3 — UI: pure property grid, not hybrid

Top pane: a pure property grid — labeled rows, editable fields for the
editable fields above, static text display for structural/display-only fields.
No more color_panel + detail text hybrid in the top pane.

Bottom pane: existing read-only monospaced detail TextCtrl, unchanged.

Splitter between top and bottom panes of the right side.

### Decision 4 — Save trigger: explicit Save button

Save button in the top (editor) pane.
- Disabled when clean (no changes since node was selected or last save).
- Enabled when dirty (any field changed).
- Dirty state is silently discarded when node focus changes — no prompt.
- On save: write changed fields to DB.

### Decision 5 — Inner sash position

Not persisted across program invocations.
Starts at 50% of the right pane height.
