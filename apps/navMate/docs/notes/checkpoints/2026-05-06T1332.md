## Leaflet visibility rendering: batching, zoom fix, nmServer restructure

---

## Session context

Continuation of T1630. That session implemented the visibility checkbox system
and identified the O(N²) Leaflet rendering problem (N separate addRenderFeatures
calls per collection toggle). This session implements the fix and several related
improvements discovered during testing.

---

## Changes made

### winDatabase.pm — accumulator batching (push and pull)

Added optional `$accumulator` parameter to `_pushObjToLeaflet`, `_renderCollection`,
and `_renderObject`. When a caller passes `\@accumulator`, features are collected
instead of calling `addRenderFeatures` immediately.

Rewrote `_pushCollectionToLeaflet` and `_onShowMap` to collect all features into
a local `@accumulator` and make one `addRenderFeatures` call at the end.

Rewrote `_pullCollectionFromLeaflet` with the same pattern: collects all UUIDs
to remove across waypoints/routes/tracks, makes one `removeRenderFeatures` call.

Result: collection-level toggle (push or pull) now makes exactly one
`addRenderFeatures` or `removeRenderFeatures` call regardless of collection size.
O(N²) → O(N).

### winDatabase.pm — GetID() crash fixed

`_onCheckboxClick` was calling `$edit_item->GetID()` to check whether the
checkbox item matched the currently-loaded editor item. `GetID()` is not bound
in this wxPerl build. Replaced with UUID string comparison:
`$this->{_edit_uuid}` vs `$node->{data}{uuid}`. Same semantics, no wxPerl
dependency.

### winDatabase.pm — double-click UX removed

Removed `_onTreeDblClick`, its `EVT_LEFT_DCLICK` binding, and the `EVT_LEFT_DCLICK`
import. Double-click on a tree label now falls through to wx default
(expand/collapse). The double-click render-to-map UX conflicts with the checkbox
visibility scheme and is no longer needed.

### nmServer.pm — shared data restructured

Replaced `$features_json :shared` (one large JSON string) with
`%features_by_uuid :shared` (uuid → per-feature JSON string).

`addRenderFeatures`: encodes new features outside the lock, then acquires
`lock($map_version)` only for fast hash assignments.

`removeRenderFeatures`: acquires `lock($map_version)` only for hash deletes.

`clearRenderMap`: acquires lock, assigns `%features_by_uuid = ()`.

`/geojson` handler: acquires lock only to copy `values %features_by_uuid`
(fast string copies), then decodes outside the lock.

Result: the wx main thread never holds `lock($map_version)` during JSON
encode/decode. Lock contention between the wx thread and the HTTP server
thread on `map_version` is eliminated.

### map.js — Leaflet zoom fix

Replaced cumulative `renderedUuids` (items ever seen) with `prevRenderedUuids`
(items present in the previous `renderAll` call). `isNew` is now computed
against the previous render, not all-time history. Toggle-off then toggle-on
correctly re-triggers zoom because the UUIDs are absent from `prevRenderedUuids`
after a removal cycle.
