### CTX_CMD notation decision

The [CTX_CMD_PASTE_NEW_AFTER] bracket notation is over-engineered. Agreed to simplify:
- Drop the CTX_CMD_ prefix
- Drop the brackets
- Use bare uppercase names: PASTE_NEW_AFTER, CTX_CMD_COPY → COPY, etc.

Wait — Patrick's exact words: "execute the PASTE_NEW_AFTER command" and "&cmd=PASTE_NEW_AFTER".
So the short form keeps only the suffix after CTX_CMD_: COPY, CUT, PASTE, PASTE_NEW,
PASTE_BEFORE, PASTE_AFTER, PASTE_NEW_BEFORE, PASTE_NEW_AFTER, DELETE_WAYPOINT,
DELETE_GROUP, DELETE_GROUP_WPS, DELETE_ROUTE, REMOVE_ROUTEPOINT, DELETE_TRACK,
DELETE_BRANCH, NEW_WAYPOINT, NEW_GROUP, NEW_ROUTE, NEW_BRANCH.

NOT the full Perl constant name, NOT the bracket notation. Just the uppercase suffix.
This applies to both the constants table (PASTE_NEW_AFTER = 10305) and all curl commands
(&cmd=PASTE_NEW_AFTER) and all prose references ("execute the PASTE_NEW_AFTER command").
