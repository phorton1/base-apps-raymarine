### §2.13 FAIL → code fix made

**Failure:** PASTE_NEW_BEFORE (cmd=10304) on a plain WP anchor (IsolatedWP2 in DST) hit
warning: `_pasteDB: PASTE_BEFORE/AFTER but no route_uuid in right_click_node` and returned
without doing anything. DB unchanged (non-catastrophic).

**Design discussion:** Patrick clarified that PASTE_BEFORE/AFTER should work for collection
members (plain WPs) as well as route points. The parent of the right-clicked node IS the
destination container -- exactly one parent, no ambiguity. For a route_point node, the
parent is the route. For a plain WP node, the parent is the collection (branch/group).
No lookup needed -- the WP's collection_uuid from the DB is the container.

**Code fix (nmOpsDB.pm):** In `_pasteDB`, replaced the 3-line warning/return block with
~50 lines that handle the collection-member case:
- Look up anchor WP from DB (getWaypoint) to get collection_uuid and position (REAL)
- Find neighbor position in same collection (SQL MAX < anchor or MIN > anchor)
- Compute midpoint; spread evenly if multiple items
- For fresh: _insertFreshWaypoint + UPDATE waypoints SET position=?
- For move: UPDATE waypoints SET collection_uuid=?, position=?
- Route_point path below is completely unchanged
