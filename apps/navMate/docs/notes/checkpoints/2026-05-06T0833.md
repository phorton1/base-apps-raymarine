## Completed work

### Schema bump: 7.0 → 8.0 (breaking, reimport required)
- `$SCHEMA_VERSION` bumped to `8.0` in `a_defs.pm`
- `visible INTEGER NOT NULL DEFAULT 0` added to all four tables:
  `collections`, `waypoints`, `routes`, `tracks`
- `sym INTEGER` dropped from `waypoints` schema
- All INSERT/UPDATE/SELECT queries in `c_db.pm` updated accordingly
- `getCollectionObjects` now returns `visible` for all three object types
- `getCollection`, `getWaypoint`, `getRoute`, `getTrack` all return `visible`

### sym removal across codebase
All `sym` references removed or replaced with hardcoded `sym => 0` at E80/KML
boundaries (deferred to future sym→wp_type mapping session):
- `c_db.pm`, `winDatabase.pm`, `nmOpsDB.pm`, `nmOps.pm`, `nmDialogs.pm`,
  `nmOldE80.pm`, `nmKML.pm` — sym removed entirely
- `nmOpsE80.pm`, `nmUpload.pm` — `sym => 0` hardcoded (E80 protocol still needs it)

### New visibility DB functions in c_db.pm (all exported)
- `getCollectionVisibleState($dbh, $uuid)` → 0/1/2 (none/all/partial) via recursive CTE
- `setTerminalVisible($dbh, $uuid, $obj_type, $visible)` — single terminal UPDATE
- `setCollectionVisibleRecursive($dbh, $uuid, $visible)` — recursive CTE UPDATE all 4 tables
- `clearAllVisible($dbh)` — UPDATE visible=0 on all 4 tables
- `getAllVisibleFeatures($dbh)` → {waypoints,routes,tracks} all visible=1, with
  route waypoints and track points attached

## Design decisions made this session (not yet in official docs)

- `visible` defaults to 0 — new imports start invisible
- Only terminal nodes (waypoints, routes, tracks) render to Leaflet; collections never render
- Tri-state on collection checkboxes computed via recursive CTE aggregate (no tree traversal)
- Collection toggle: bulk DB write to all descendants, Leaflet delta via getCollectionWRGTs
- Clear Map: `visible=0` across all 4 tables, atomic, from menu OR Leaflet button
- sym → 0 hardcoded at E80 boundary; full sym→wp_type mapping deferred to later cycle
- Global memory updated: `conventions_wxperl.md` — editor layout convention (absolute positioning,
  no FlexGridSizer, no intermediate containers, named constants, EVT_SIZE for long-string only)
