## What happened this session (after T1300 checkpoint)

### Bug discussion and code fixes

Worked through the §3.0c (route paste with 0 route_waypoints) bug in depth.
Key conclusions from analysis:

1. SS = Specification Section -- the numbered rules in nmOperations.md itself.

2. SS10.10 was NOT implemented anywhere -- not in getPasteMenuItems, not in
   _doPaste pre-flight, not in _pasteRouteToE80. Design rule existed; no code.

3. Both _pasteRouteToE80 AND _pasteNewRouteToE80 had the same {members} key bug.
   Route clipboard items are built with key route_points (nmOps.pm:298); both
   paste functions read {members}, which is absent on route items, so the // []
   fallback always fired and the WP loop ran zero times.

4. The {members} bug in _pasteRouteToE80 was masking two missing implementations
   simultaneously: the SS10.10 route dependency check, and the route sequence
   assembly. Both were invisible because the loop never ran.

5. Fixing the key alone without implementing SS10.10 would have caused
   _pasteRouteToE80 to auto-create member WPs on E80 -- violating the design
   rule. SS10.10 explicitly states navMate does not silently paste missing WPs.

### Fixes applied (authorized by Patrick)

**nmOpsE80.pm -- _pasteRouteToE80 (rewritten):**
- Reads route_points not members
- Replaced WP-creation loop with SS10.10 existence check:
  if any route_point UUID not in wpmgr->{waypoints}: error IMPLEMENTATION ERROR + return
- Progress total = 1 (just the route create/modify; no WP operations)
- Removed per-WP cut logic (route cut still handled at route level)

**nmOpsE80.pm -- _pasteNewRouteToE80:**
- Line 1123: reads route_points not members (variable still named $members internally)
- Progress total stays scalar(@$members)+1 which now correctly counts route_points

**nmOpsE80.pm -- _pasteAllToE80 count:**
- Route total changed from scalar(@$members)+1 to 1
  (UUID-preserving paste: WPs already on E80, only route op counts)

**nmOpsE80.pm -- _pasteNewAllToE80 count:**
- Route total changed to scalar(@{$item->{route_points}//[]})+1
  (fresh-UUID paste: N WP creates + 1 route create)

**nmOps.pm -- buildContextMenu:**
- Added SS10.10 menu guard: after getPasteMenuItems, if panel=e80 and
  any route in clipboard has a member WP UUID absent from wpmgr->{waypoints},
  remove PASTE (not PASTE_NEW) from @paste.
  Variable named $cb_item (not $item) to avoid shadowing outer scope.
