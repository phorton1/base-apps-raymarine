## showError wiring correction (done this session)

Original implementation put `showError` override in winMain.pm — wrong on two counts:
(a) Wrong class: navMate-specific base behavior belongs in w_frame.pm
(b) Not overridable: Pub::Utils::error() called Pub::WX::Frame::showError() as a
    static package function, bypassing Perl OO dispatch entirely.

Fix applied:
1. `Pub::Utils.pm` line 741: changed static call to polymorphic with ref guard:
   `ref($use_frame) ? $use_frame->showError(...) : Pub::WX::Frame::showError(...)`
2. `w_frame.pm`: added `use nmDialogs;` + `showError` override checking
   `$nmDialogs::suppress_error_dialog`
3. `winMain.pm`: rebased from `Pub::WX::Frame` to `w_frame`; removed bad override

## Bug found and fixed: $dbg_e80_ops uninitialized

**Symptom**: Perl warning visible in cmd.exe console (NOT in ring buffer):
`Use of uninitialized value $level in numeric le (<=) at /base/Pub/Utils.pm line 713`
Appeared between ProgressDialog STARTED and first `_pasteGroupToE80` log line.

**Cause**: nmOpsE80.pm line 19 declared `our $dbg_e80_ops;` with comment
"declared in nmOps.pm" — but nmOps.pm never initialized it. All display() calls
in nmOpsE80.pm using `$dbg_e80_ops` as the level arg passed undef.

**Fix**: Added `our $dbg_e80_ops = 0;` to nmOps.pm alongside `our $dbg_ops = 0;`.
