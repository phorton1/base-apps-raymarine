## Tree expand button fix ‚Äî root cause and resolution

### Problem observed

The wxTreeCtrl expand buttons (+/-) were unreliable after adding the visibility
checkbox images. Some folders would not expand on click, with zero visual
feedback. The behavior was inconsistent across items at the same indentation
level and was not present before the visibility work.

### Diagnostic process

Two hypotheses were tested by commenting out code incrementally:

1. `EVT_LEFT_DOWN` interception (our handler) ‚Äî commented out. Helped slightly
   but the expand issue persisted, ruling it out as the sole cause.
2. `SetImageList` + all `SetItemImage` calls ‚Äî commented out. Expand buttons
   became completely solid and a subtle item-label flyover/highlight effect
   also disappeared. This was the root cause.

### Root cause

Adding **any** normal image list (`SetImageList`) to a Windows native TreeCtrl
activates a different rendering mode in the Win32 TreeView control ‚Äî hot-tracking
is enabled, item layout shifts slightly, and the expand button hit area becomes
unreliable. This is a native Win32 TreeView behavior, not a wxWidgets bug.

### Solution

Use `SetStateImageList` instead. State images occupy a dedicated slot between
the expand button and the normal icon, with their own hit-test zone
(`wxTREE_HITTEST_ONITEMSTATEICON`). This is the correct wxWidgets mechanism for
per-item checkboxes on a tree and does not perturb the expand button geometry.

### wxPerl gap (implementation detail ‚Äî do not record in official or working docs)

`SetStateImageList` and `wxTREE_HITTEST_ONITEMSTATEICON` were already present in
Patrick's wxPerl. Only `SetItemState` and `GetItemState` were missing from the
XS bindings. Two small XS stubs (following the identical pattern of existing
methods) were added to `/src/wx/wxPerl/XS/TreeCtrl.xs` and wxPerl was rebuilt
(`dmake` + `dmake install` from `/src/wx/wxPerl`). The build was clean.
Update Claude: acknowledge this happened; do not record it anywhere.

### winDatabase.pm changes

- `SetImageList` ‚Üí `SetStateImageList`
- `SetItemImage($item, N)` ‚Üí `SetItemState($item, N)` throughout (numbering
  unchanged ‚Äî wxWidgets adjusts the Win32 1-based offset internally)
- `GetItemImage($item)` ‚Üí `GetItemState($item)`
- `wxTREE_HITTEST_ONITEMICON` ‚Üí `wxTREE_HITTEST_ONITEMSTATEICON`
- `EVT_LEFT_DOWN` re-enabled (was temporarily disabled during diagnostics)

### Result

Expand buttons solid. Visibility checkboxes functional. No flyover/hot-tracking
side effects observed.
