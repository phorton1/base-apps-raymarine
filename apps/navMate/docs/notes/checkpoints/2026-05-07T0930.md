## Bug fixes applied

### winDatabase.pm — Branch labeled "Group" in editor pane
Operator precedence bug on line ~1097. `$data->{node_type} // '' eq $NODE_TYPE_GROUP`
was parsed as `$data->{node_type} // ('' eq $NODE_TYPE_GROUP)` — `//` binds lower than
`eq` in Perl. Since node_type is always defined, the condition was always truthy → always
"Group". Fixed by parenthesizing: `($data->{node_type} // '') eq $NODE_TYPE_GROUP`.

### c_db.pm + winDatabase.pm — Tree summary showed "folder" for groups
`getCollectionCounts` returned a single `collections` count with no type breakdown.
Modified it to query groups and branches separately, returning `groups`, `branches`, and
`collections` (sum) keys. Updated `_collectionLabel` in winDatabase.pm to use `groups`
and `branches` separately — groups now say "group/groups", branches still say
"folder/folders". The `collections` key is preserved so `_addCollectionItem`'s
dummy-node check is unaffected.

### Em-dash encoding — systemic replace across navMate, shark, NET
41 UTF-8 em-dash bytes (0xE2 0x80 0x94) replaced with ASCII hyphen (`-`) across 12 .pm
files. Used `:raw` I/O to preserve line endings. Files touched:
- apps/navMate: a_utils.pm, c_db.pm, navMate.pm, nmKML.pm, winDatabase.pm,
  winE80.pm, winMain.pm, winMonitor.pm
