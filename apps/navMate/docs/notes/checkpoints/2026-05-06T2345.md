## Corrections applied

### CTX_CMD constants (§8.3)
- Collapsed 10 type/branch-specific COPY/CUT constants to just `COPY = 10010` and `CUT = 10110`.
- The menu label is always "Copy" / "Cut" — no type-qualified variants.

### Destination-side model (§2, §3)
- Core design principle established: **intelligence lives at the destination, not the source.**
- Copy/Cut places no restriction at source. The clipboard is a dumb container.
- Pre-flight runs **before the paste context menu is built**, evaluating effective contents.
- Branch items **dissolve** at E80 pre-flight: their contents float up and are evaluated
  as if held directly. A branch containing only waypoints → E80 paste offered to waypoint
  destinations. A branch with mixed contents → no E80 paste offered.
- §3 clipboard fields: removed "Selection origin"; query sequence now leads with dissolution.

### Branch clipboard (§2.1, §2.2, §6.2)
- Branch is NOT special at copy time. Copy is always offered; branch is just an item.
- Branch has no fixed "no E80 destination" rule. E80 pre-flight dissolves it and evaluates.
- §6.2 table updated: branch row shows compatible E80 destination based on dissolved type.

### Route points (§2.6, §2.3, §6.3)
- Route points ARE first-class clipboard objects. Copy/Cut produce clipboards of ordered
  UUID references.
- Key insight: a waypoint and a route point are both waypoint UUID references at the route
  level. The type distinction dissolves at route insertion.
- A clipboard of waypoints, route points, or a **mix of both** is valid for Paste
  Before/After into a route on both DB and E80 sides.
- §6.3 tables updated with route point rows and mixed waypoints+route points rows.

### Selection model (§2.1)
- Same-parent selection was proposed, prototyped in discussion, then rejected as too
  complex to specify. The section was removed entirely rather than left as "deferred."
- The tree is free. Copy/Cut handlers define their own acceptance rules.

### Delete (§4.3)
- Simplified to two menu items: **Delete** and **Delete Group + Waypoints**.
- Confirmation dialog presents type and quantity. Route reference dependencies surfaced
  at confirmation time, not as menu items.
- DELETE_* CTX_CMD constants (§8.3) are all retained — the single "Delete" menu item
  dispatches different constants based on right-click node type.
- DELETE_GROUP (dissolve): members survive on both sides (float up to parent collection
  on DB; float up to My Waypoints on E80). No route reference check needed.
- DELETE_GROUP_WPS: route reference check applies to members on both sides.

### User-level types (§3)
- Branch added as a fifth user-level type alongside waypoint, route, track, group.
- Branch is DB-only; has no E80 counterpart.
