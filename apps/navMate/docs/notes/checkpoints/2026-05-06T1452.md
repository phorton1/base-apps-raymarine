## Show on Map / Hide on Map — reworked (winDatabase.pm)

`_onShowMap` and `_onHideMap` in winDatabase.pm were disconnected from the
persistent visibility checkbox scheme. They have been replaced with a new
implementation that writes to the DB and syncs checkboxes.

### What changed

`_onShowMap` and `_onHideMap` are now one-liner wrappers calling `_onShowHideMap($this, 1/0)`.

New subs added:
- `_onShowHideMap` — batch visibility setter; sets `visible` in DB, updates
  tree checkbox states, pushes/pulls Leaflet, syncs editor checkbox
- `_analyzeShowHideSelection` — classifies tree selection into case1 collections,
  case2 collections, and leaf nodes
- `_hasSelectedDescendant` — recursive tree widget walk used by the classifier

Dead code removed: `_renderCollection` and `_renderObject` (toggle-render path
that predated the visibility checkbox scheme).

### Branch Case 1 / Case 2 logic (design decision confirmed)

When a branch/collection appears in a multi-selection:
- **Case 1**: no descendants of that branch are in the selection →
  treat as deliberate target → `setCollectionVisibleRecursive` (same as checkbox)
- **Case 2**: at least one descendant is in the selection →
  branch was "caught up" incidentally → skip the branch; only the explicitly
  selected descendants are touched; branch checkbox state is recomputed as derived

Detection is tree-widget-side only (no DB queries); only loaded/visible nodes
can be in a selection, so this is correct and fast.
