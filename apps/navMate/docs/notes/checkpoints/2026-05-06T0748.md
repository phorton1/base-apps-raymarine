## Session 2026-05-06 — editor panel refactor in winDatabase.pm

### What was done

Refactored the editor panel in `winDatabase.pm` from a `Wx::FlexGridSizer` layout
with intermediate sub-panels to absolute positioning driven by named constants.

**Constants added** (all at top of editor construction block):
```
$ED_MARGIN, $ED_LABEL_W, $ED_COL_GAP, $ED_CTRL_X
$ED_CTRL_H, $ED_ROW_GAP, $ED_ROW_H
$ED_HEADER_SIZE, $ED_BOTTOM_MARGIN, $ED_MAX_ROWS
$ED_INITIAL_SASH = $ED_HEADER_SIZE + $ED_MAX_ROWS * $ED_ROW_H + $ED_BOTTOM_MARGIN
```

**Layout changes:**
- Save button moved to header row, label column: `[$ED_MARGIN, $ED_MARGIN]`, width `$ED_LABEL_W`
- Bold type title moved to header row, control column: `[$ED_CTRL_X, $ED_MARGIN]`
- All 4 intermediate sub-panels removed (lat_panel, lon_panel, color_row, depth_panel)
- All controls placed directly on `$editor_panel` at computed absolute positions
- `EVT_SIZE` on editor_panel resizes only `ed_name` and `ed_comment` (long string controls)
- `_ed_show_row` simplified: no sizer arg, just `->Show()` on label and control
- `_clearEditor` / `_loadEditor` updated to show/hide companion controls individually
  (ed_lat_ddm, ed_lon_ddm, ed_color_swatch, ed_pick_btn, ed_depth_unit)

**Sash change:**
- `SplitHorizontally` now passes `$ED_INITIAL_SASH` instead of 0
- `SetSashGravity` changed from 0.5 to 0 (editor pane stays fixed on window resize)

**Global memory updated:**
- `C:\Users\Patrick\.claude\global\conventions_wxperl.md` — added editor panel
  layout convention: no FlexGridSizer, no intermediate containers, absolute
  positioning with named constants, EVT_SIZE only for long-string controls.
