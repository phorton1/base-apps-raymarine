## What was implemented this session (Phases 5 and 6)

### Phase 5 — Leaflet startup push (browser-connect event)

**nmServer.pm:**
- Added `$browser_connect_event :shared = 0`
- In `/poll` handler: if `time() - $last_poll_time >= 3` (browser was disconnected),
  increments `$browser_connect_event`. Fires on first connection at startup (since
  `$last_poll_time` initializes to 0).
- Added `pollBrowserConnectEvent()` — returns 1 and resets the flag; 0 otherwise
- Exported `pollBrowserConnectEvent`

**winMain.pm:**
- In `onIdle`: calls `pollBrowserConnectEvent()`; if true, finds the winDatabase
  pane and calls `$database->onBrowserConnect()`

**winDatabase.pm:**
- `sub onBrowserConnect`: calls `clearRenderMap()`, resets `$last_clear_version`
  and `%rendered_uuids`, fetches all `visible=1` features via `getAllVisibleFeatures`,
  builds GeoJSON for waypoints/routes/tracks (using pre-fetched waypoint/point
  arrays from the DB result), calls `addRenderFeatures`. Routes use
  `$r->{waypoints}` from the getAllVisibleFeatures result; tracks use `$t->{points}`.

### Phase 6 — Clear Map command

**nmServer.pm:**
- Added `$clear_map_pending :shared = 0`
- In `/clear` handler: calls `clearRenderMap()` immediately (so browser sees empty
  map at next poll), then sets `$clear_map_pending = 1` to signal wx thread
- Added `pollClearMapPending()` — returns 1 and resets; 0 otherwise
- Exported `pollClearMapPending`

**w_resources.pm:**
- Added `$COMMAND_CLEAR_MAP = 10030`, exported it
- Added to `$command_data`: label "Clear Map", description "Set all visible=0 and
  clear the Leaflet map"
- Added to `$view_menu` immediately below `$COMMAND_OPEN_MAP` (no separator between)

**winMain.pm:**
- Added `EVT_MENU($this, $COMMAND_CLEAR_MAP, \&onCommand)` binding
- In `onIdle`: calls `pollClearMapPending()`; if true, calls `$database->onClearMap()`
- In `onCommand`: `elsif ($id == $COMMAND_CLEAR_MAP)` → finds database pane, calls
  `$database->onClearMap()`

**winDatabase.pm:**
- `sub onClearMap`: connects DB, calls `clearAllVisible($dbh)`, disconnects,
  calls `clearRenderMap()`, resets `$last_clear_version = getClearVersion()`,
  clears `%rendered_uuids`, calls `$this->refresh()` to reload tree with all
  checkboxes shown as unchecked

## Design decisions confirmed this session

- `visible` defaults to 0 — new imports start invisible
- Only terminal nodes (waypoints, routes, tracks) render to Leaflet; collections
  never render directly
- Tri-state on collection checkboxes computed via recursive CTE aggregate
- Collection toggle: bulk DB write to all descendants, Leaflet delta update
- Browser reconnect: always clear + re-push visible features for clean sync
- Clear Map: `visible=0` across all 4 tables, atomic, from menu OR Leaflet button
- sym → 0 hardcoded at E80 boundary; full sym→wp_type mapping deferred
