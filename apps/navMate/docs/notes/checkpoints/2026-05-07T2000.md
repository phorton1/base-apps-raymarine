### onContextMenuCommand($cmd_id, $panel, $right_click_node, $tree, @nodes)
The main dispatch function. Pattern from old nmClipboard.pm::onContextMenuCommand:
```perl
display(-1, 0, "===== $label ($panel) STARTED =====", 0, $UTILS_COLOR_LIGHT_MAGENTA);
# dispatch to pre-flight + execution
display(-1, 0, "===== $label ($panel) FINISHED =====", 0, $UTILS_COLOR_LIGHT_MAGENTA);
```

Dispatch table:
- CTX_CMD_COPY → _doCopy($panel, $right_click_node, @nodes)
- CTX_CMD_CUT  → _doCut($panel, $right_click_node, @nodes)
- PASTE family (10300-10305) → _doPaste($cmd_id, $panel, $right_click_node, $tree, @nodes)
- DELETE family (10410-10450) → _doDelete($cmd_id, $panel, $right_click_node, $tree, @nodes)
- NEW family (10510-10550) → _doNew($cmd_id, $panel, $right_click_node, $tree)
- Unknown → warning(0, 0, "nmOps: unknown cmd_id=$cmd_id")

### _doCopy / _doCut
Pre-flight: non-empty selection (SS9).
Snapshot each node from the tree into clipboard items.
Call setCopy/setCut with source panel + items array.

Node snapshot format:
```perl
{ type => 'waypoint'|'group'|'route'|'track'|'route_point',
  uuid => '...',
  data => { ...node data hash... } }
```
For groups: add `members => [...]` (array of member WP snapshots).
For routes: add `route_points => [...]` (ordered WP UUID refs).
For route_points: add `route_uuid => '...'` and `position => N`.

### _doDelete($cmd_id, $panel, $right_click_node, $tree, @nodes)
Pre-flight per SS8. Then dispatches to nmOpsDB::_deleteDB or nmOpsE80::_deleteE80.

### _doPaste($cmd_id, $panel, $right_click_node, $tree, @nodes)
Pre-flight per SS10 (the full resolution sequence):

**Step 1** — Ancestor-wins resolution (SS6.2).
Walk `$clipboard->{items}`, build resolved list removing descendants of any ancestor present.
If resolution changed the list: confirmDialog (ancestor-wins warning, list what was absorbed).
If resolved list is empty: error + return.

**Step 2** — Empty guard (belt-and-suspenders).

**Step 3** — Recursive paste check (SS1.5).
Walk destination ancestry upward. Reject if any ancestor UUID matches any resolved item UUID.

**Step 4 (E80 only)** — Branch dissolution (SS6.3).
Replace Branch nodes in resolved list with their DB descendants at native types.

**Step 5 (E80 only)** — Homogeneity check.
If effective contents heterogeneous (not mixed-wp-and-rp): error + return.

**Step 6 (E80 only)** — Intra-clipboard name collision.
Check for same-type same-name within effective contents. Hard abort if found.

**Step 7 (E80 only)** — E80-wide name collision.
Check each item name against WPMGR in-memory data. Hard abort if found.

After pre-flight passes: dispatch to nmOpsDB::_pasteDB or nmOpsE80::_pasteE80.

### _doNew($cmd_id, $panel, $right_click_node, $tree)

## Key invariants / behavioral rules (from 1900 checkpoint, kept here for convenience)

- NEVER `die`/`kill` in .pm files — use `error("msg"); return;`
- `warning(level, indent, msg)` — three args; `error(msg)` — one arg
- Never Write tool on pre-existing files (changes line endings)
- `use` order: pure Perl → Wx/Pub → NET → own local → use base
- Always explicit qw() imports from Pub::Utils
- Schema guard: confirm before any SQLite schema/structure change
- No polling loops; no Yield() in polling
- $progress, never $prog, for progress object variables
- DB-cut-to-E80 is NEVER valid (pre-flight rejects at paste time)
- Route dependency check before E80 route paste (SS10.10)
- Execution guarantee: once pre-flight passes, run to completion
