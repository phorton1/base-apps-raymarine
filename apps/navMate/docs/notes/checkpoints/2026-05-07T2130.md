**Private helpers:**

**buildContextMenu($panel, $right_click_node, @nodes):**
Calls nmClipboard menu generators (getDeleteMenuItems, getNewMenuItems,
getCopyMenuItems, getCutMenuItems, getPasteMenuItems), assembles into a Wx::Menu
with separators. Returns Wx::Menu. Does NOT bind EVT_MENU - that's the caller's job
(winDatabase/winE80 do that in their new() via allCopyCmds/allCutCmds/etc).

**onContextMenuCommand($cmd_id, $panel, $right_click_node, $tree, @nodes):**
Display-log framing around dispatch. Range-based dispatch:
- 10010 (COPY) -> _doCopy
- 10110 (CUT) -> _doCut
- 10300-10305 (PASTE family) -> _doPaste
- 10410-10450 (DELETE family) -> _doDelete
- 10510-10550 (NEW family) -> _doNew

**_doCopy / _doCut:**
Calls _snapshotNodes($panel, @nodes). If snapshots succeed, calls setCopy/setCut.
Cut is just a flagged copy - no immediate deletion. (See todo [Item 11] for that design Q.)

**Snapshot system:**
`_snapshotNodes` dispatches to `_snapshotDBNode` or `_snapshotE80Node` per item.

`_snapshotDBNode` handles:
- `type='route_point'` -> {type, uuid, route_uuid, position, data}
- `type='object', obj_type='waypoint'` -> getWaypoint; {type, uuid, data}
- `type='object', obj_type='route'` -> getRoute + getRouteWaypoints; adds route_points=[]
- `type='object', obj_type='track'` -> getTrack + getTrackPoints; puts pts in data.points
- `type='collection', node_type=group` -> getCollection + getGroupWaypoints; members=[]
- `type='collection', node_type=branch` -> getCollection + _snapshotBranchContents; type='branch'

`_snapshotBranchContents` recursively calls getCollectionChildren + getCollectionObjects,
snapshots each via _snapshotDBNode.

`_snapshotE80Node` handles:
- route_point -> _e80WpClipData applied
- waypoint -> _e80WpClipData applied
- group -> data + members (each WP scaled)
- my_waypoints -> type='group', uuid=undef, name='My Waypoints', members=ungrouped WPs scaled
- route -> data + route_points (ordered by uuids array, each WP scaled)
- track -> data as-is (E80 tracks don't have point arrays in memory)

**Clipboard item format (new canonical):**
```
{ type => 'waypoint'|'group'|'route'|'track'|'route_point'|'branch',
  uuid => '...',
  data => { ... },
  # type-specific extras:
  members     => [...]   # for group, branch
  route_points => [...]  # for route
  route_uuid  => '...'   # for route_point
  position    => N       # for route_point (DB only; E80 uses array index)
}
```

Note: 'branch' is not in the checkpoint spec's type list but IS needed because:
(a) getCopyMenuItems allows copying DB collections (which includes branches)
(b) _doPaste pre-flight step 4 (branch dissolution) replaces branches with their members

**_doDelete pre-flight:**
Branch-safety check (isBranchDeleteSafe) inline for DB branches.
Waypoint route-ref check deferred to nmOpsDB::_deleteDB per-waypoint.
Dispatches to nmOps::_deleteDB or nmOps::_deleteE80 (stubs in the two stub files).

**_doPaste - full pre-flight:**
Step 1: _resolveAncestorWins - walks items, skips any whose uuid appears as a member
        of another item. If list shrinks, shows confirmDialog gated by
        $nmDialogs::suppress_confirm. Returns if user rejects.
Step 2: Empty guard - error if nothing remains.
Step 3: Recursive paste check - if dest_uuid is found in resolved items, error.
Step 4 (E80 only): Branch dissolution - replace branch items with their members array.
Step 5 (E80 only): Homogeneity check - error if mixed routes+tracks with waypoints
        (allows waypoint+group together as that's valid).
Step 6 (E80 only): Intra-clipboard name collision - same type, same name = error.
Step 7 (E80 only): E80-wide name collision - checks wpmgr in-memory for wp/group/route.
Then dispatches to nmOps::_pasteDB or nmOps::_pasteE80 (stubs).

D-CT-DB check: if panel=e80 AND cut_flag=1 AND source='database' -> hard reject at top.

**_doNew:**
NEW BRANCH / NEW GROUP (database panel): inline Wx::TextEntryDialog -> insertCollection.
NEW GROUP (e80): stubs to nmOps::_newE80Group.
NEW ROUTE / NEW WAYPOINT: stubs to nmOpsDB/nmOpsE80 sub-file.

## Key design decisions made this session

- **buildContextMenu returns Wx::Menu** (not {id,label} arrays) - confirmed by reading
  both winDatabase.pm and winE80.pm, which both pass _buildContextMenu result to PopupMenu.
- **'branch' added as clipboard item type** - getCopyMenuItems allows it; _doPaste step 4
  dissolves it. Not in original checkpoint spec type list but logically required.
- **_doNew branch/group inline** - Wx::TextEntryDialog inline in nmOps.pm rather than
  depending on nmDialogs::showNewCollection (whose existence is not confirmed). Clean,
  matches what winDatabase.pm currently does, no new dependency.
- **Cut = snapshot + setCut only** - no immediate deletion. Design Q deferred (todo [Item 11]).
- **nmDialogs::suppress_confirm accessed as $nmDialogs::suppress_confirm** (not imported)
  to avoid circular dependency concerns.

### _deleteDB($cmd_id, $right_click_node, $tree, @nodes)
Dispatches on cmd_id:
- CTX_CMD_REMOVE_ROUTEPOINT -> remove WP from route (UPDATE route_waypoints)
- CTX_CMD_DELETE_WAYPOINT -> for each node: check route refs (isBranchDeleteSafe or custom check); confirmDialog if refs exist; deleteWaypoint
- CTX_CMD_DELETE_GROUP -> deleteCollection (groups only)
- CTX_CMD_DELETE_GROUP_WPS -> deleteCollection + deleteWaypoint for each member WP
- CTX_CMD_DELETE_ROUTE -> deleteRoute
- CTX_CMD_DELETE_TRACK -> deleteTrack
- CTX_CMD_DELETE_BRANCH -> deleteBranch (safety already checked in _doDelete pre-flight)
After all deletes: call _refreshDatabase(); call onObjectsDeleted for Leaflet sync

### _pasteDB($cmd_id, $right_click_node, $tree, $items, $cb)
$items is the resolved list (already ancestor-wins cleaned, from _doPaste pre-flight).
$cmd_id determines paste positioning (PASTE/PASTE_NEW/PASTE_BEFORE/PASTE_AFTER/etc).
For each item by type:
- waypoint -> insertWaypoint under target collection (or conflict dialog if name collision)
- group -> insertCollection(GROUP) + insertWaypoint for each member
- route -> insertRoute + insertRouteWaypoint for each route_point
- track -> insertTrack + insertTrackPoints
- branch -> insertCollection(BRANCH) + recurse into members

### _newDatabaseWaypoint($right_click_node, $tree)
